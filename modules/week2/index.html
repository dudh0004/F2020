# 1. Everything is a file

Your Mac uses macOS and macOS is UNIX. In UNIX everything is represented by a file: documents, directories, serial ports, shell scripts, hard drives, sockets, printers, everything! This has great implications on how macOS manage its tasks. As your knowledge and experience increases, you will see some of the benefits of this aproach.

Because in macOS everything is represented by a file it is very important you can efficiently manipulate files. You will spend a great amount of time handling files: creating, copying, renaming, archiving, joining and so on.

But before we even start to do that we need to understand some other important concepts: absolute and relative paths, wildcards and file permissions. Once these concepts are known, we can make a clever use of file handling in macOS.

# 2. Absolute path vs. Relative path

The filesystem is a data structure used to keep track of files on a disk or partition; that is, the way the files are organized on the disk. Think of it as a tree: there is a root/trunk and branches (directories) that flow from it.

The file system is rooted, as you can imagine, at the root directory: /

There are directories at the root: /home, /Library, /usr and others.

These directories have sub-directories of their own, and so on. That is how the filesystem is laid out. When specifying file paths, you can specify a directory in terms of the root: /Users/lennonj/temp, /usr/bin, /Library/Image\ Capture/Support. In this case we say this is a absolute path.

TIP

Absolute paths always start with forward slash: /

We can also specify a path relative to another directory. So, one dot slash ./ means the current working directory and ../ means the parent directory.

TIP

  • ../../ means "up two levels" i.e. the parent of the parent.
  • ../../../ means "up three levels" and so on.

# 3. Wildcards

Instead of specifying individual files, you can run a command on any file that matches a pattern. This saves you quite a lot of time since you don't have to work on each file separetely. To create those patterns we make use of wildcards. There are three types of wildcards:

# Star

The star * stands for zero or more characters.

Example 1

v* will match every file with the base name starting with v

  • 👍 Will match

    • v
    • v_
    • victory
    • v_drives_not_to_be_used.sys

  • 👎 Will not match

    • anything.txt
    • my_file.virtual
    • _v
    • .virtual.cfg

Question

Does v* match Victor.txt?

Example 2

phi* will match every file with the base name starting with phi

  • 👍 Will match

    • phi
    • phi-
    • philharmonic

  • 👎 Will not match

    • physician.txt
    • 1philosophy.doc
    • Philately.txt

Question

Does phi* match philanthropist.alt?

Example 3

*land will match every file ending in land

  • 👍 Will match

    • parkland
    • Parkland
    • _disneyland
    • _q_W_e_R_t_Y_land

  • 👎 Will not match

    • LalaLand
    • Wonderland.doc

Question

  • Does *land match my.land?
  • Does *.land match my.land?
  • Can you come up with another expression that uses * and matches my.land?

Example 4

*.txt will match every file with extension .txt

  • 👍 Will match

    • work.txt

  • 👎 Will not match

    • txt
    • _txt

Question

Does *.txt match .txt?

# Question mark

The question mark ? stands for a single character

Example 1

week?.md

  • 👍 Will match

    • week0.md
    • week1.md
    • weekA.md

  • 👎 Will not match

    • week.md
    • week1.txt
    • week10.md
    • wee4.md

Question

Does result?.json match both results.jsonand result0.json?

Example 2

week??.md

  • 👍 Will match

    • week10.md
    • week1_.md
    • weekA1.md

  • 👎 Will not match

    • week.md
    • week01.txt
    • week1.md
    • wee14.md

Question

Does photo???.jpg match both photo123.jpeg and photo331.jpeg?

Example 3

?sep20??.md

  • Will match

    • 1sep2021.md
    • asep2021.md
    • _sep2021.md

  • 👎 Will not match

    • 1sep2021.mb
    • 01sep2021.md
    • _1sep2021.md
    • sep2021.md

Question

Does ??.doc match _1.doc?

# Square bracket

Square brackets bound a group of single characters to match. For example, week[12].md would match week1.md or week2.md and [Ww]eek01.md matches either Week01.md or week01.md. You can use a hyphen - to separate a range of consecutive characters. For example, week[1-3].md would match week1.md, week2.md, or week3.md.

Example 1

[Ww]eek0[125].md

  • 👍 Will match
    • week01.md
    • Week02.md
    • week05.md
  • 👎 Will not match
    • Week0125.md
    • Week0.md

Example 2

Week0[125][0-9].md

  • 👍 Will match
    • Week011.md
    • Week021.md
    • Week052.md
    • Week025.md
  • 👎 Will not match
    • Week01.md
    • Week25.md
    • Week025
    • Week0229.md

Questions

  • Does [Mm]onths.20[09][09].json match Months2000.json?
  • Does [Mm]onths.20[09][09].json match Months.2021.json?
  • Does [Mm]onths.20[09][09].json match Months.2099.json?
  • Does [Mn]onths.20[09][09].json match months.2099.json?
  • Does [Mm]onths.20[0-9][0-9].json match Months.2056.json?

# 4. File permissions

Files have different permissions for reading, writing and running.

File modes

  • r is the flag for readable
  • w is the flag for writable
  • x is the flag for executable

References are used to distinguish to whom the permissions apply. If no classes are specified all is implied.

References (Classes of users)

  • u is the reference for user. The file owner.
  • g is the reference for group. Members of the file's group.
  • o is the reference for others. Users who are neither the file's owner nor members of the file's group.
  • a is the reference for all three of the above, same as ugo.

Example

-rwxr--r--  1 vladimircezar  staff   0  3 Sep 21:23 script.sh
1

The line above shows a file in the filesystem.

The 1st character - indicates it is a file (and not a d directory, for example).

The 2nd, 3rd and 4th characters, indicate this file is readable r, writable w and x executable by the user.

The 5th, 6th, and 7th characters indicate this file is readable r (but not writable or executable) by all other users in the group staff.

The 8th, 9th and 10th characters indicate this file is readable r (but not writable or executable) by all other users.

We can change these permission attributes both by using the Graphical User Interface or through the Command Line Interface.

# 5. Terminal

The Graphical User Interface, by the use of windows and point and click devices like a mouse give us the ability to interact with macOS. Another way to wotk with the operating system is through the command line. We use a special program, the Terminal, to do that.

Press ⌘ + Space. It will bring the Spotlight search bar. Type Terminal. Click on the Terminal application to launch the command line terminal.

# 6. Command Line Interface

# pwd

Sometimes, specially if you are new to the command line, it is easy to get lost i.e you don't know what is the working directory. To help you find out where you are, in Terminal, type pwd and hit enter (⏎) to “print the working directory”.

To change directories, type cd for change directory followed by the directory to go to.

# cd, ls and md

From the home directory, type cd ... That’s cd followed by a space, followed by two dots. You want to change directory up (represented by ..). Every directory has a parent (up) directory, except for the root ( written as /).

TIP

cd ~ ⏎ and or only cd ⏎ goes to your home directory ( /Users/your_name )

Now, if you type pwd, you should be in /Users. To see all the files and folders where you are, type ls for list. You should see “guest”, and your name. To go back to your home directory, type cd your_name. If you type pwd now, you should see /Users/your_name.

Don`t forget

  • On the command line, the first word is always the command.
  • Parameters are everything that comes after the command and affect the command.
  • Spaces separate commands from the parameters.

From your home directory, type mkdir temp. The command is mkdir (make directory), and the parameter is temp (what directory to make).

Type ls (list) to see what files and folders are in your directory. You should now see temp since you just created it. How do we visit your new directory? cd temp for change directory to temp.

Within /Users/yourname/temp, type ls (list). You should only see . (current directory), and .. (up one directory). To go up a directory, type cd ... If you’ve gone up one directory, how do we check? pwd

# Copying files

  1. Move to temp directory: cd ~/temp
  2. Make a file by typing echo “Hello World” >> file.txt
  3. The >> means send the text to the file.
  4. To view the contents of a file, type more file.txt
  5. Type ls to see the files in the directory.
  6. Type cp file.txt file2.txt. This makes a copy of the file, and calls it file2.txt. The command line supports auto-complete. Try typing the first few letters of file.txt and hit tab to auto-complete the file name.

# Moving files

  1. Go to the temp directory: cd ~/temp

  2. Type ls to see what files are in the temp directory. There should be temp.txt and temp2.txt.

  3. You can rename a file using the move command. Type mv temp.txt temp1.txt. This means move temp.txt to the file temp1.txt.

  4. Type ls to see the files in the directory.

  5. Type mv temp1.txt ../temp1.txt. This means move temp1.txt up one directory (../)

  6. The file temp1.txt is now up one directory. You can move it back by typing mv ../temp1.txt temp1.txt. This means look up 1 directory (../) for a file called temp1.txt, and move it into the current directory.

  7. How would you move the temp1.txt file from the parent directory to a new name (newfile.txt) but still in the parent directory?

    mv ../temp1.txt ..newfile.txt
    1

# Deleting files

  1. You can delete files using the rm command. This command only takes one parameter, the file to delete: rm file2.txt
  2. This does not send a file to the trash bin, it deletes it immediately. There is no undoing this!!
  3. To create a directory, use mkdir (make directory)
  4. To remove a directory, use rmdir (remove directory), but the directory must be empty

# 7. Hidden files (again)

macOS has the concept of hidden files. These are files that don’t normally show up in a folder window, or are listed by the ls command. A file is hidden if is starts with a dot ..

Use mv to rename a file so that it’s hidden: mv file2.txt .file2.txt

This renames the file file2.txt to .file2.txt. If you type ls, you won’t see this file anymore. To see hidden files, you have to type ls –a . This means list all files, including the hidden ones. In finder, type cmd, shift and . to see hidden files.

You can also have hidden directories.

The system’s trash folder is a hidden directory in your home directory: ~/.Trash

# 8. Changing permissions

Type chmod u=rwx File1.txt. This means the user can read, write, and execute the file.

If you can execute a file, you don’t need the sh command to process the file. You can just type ./File1.txt and it will execute the file.

Try chmod u=x File1.txt. This means it doesn’t have read or write permissions. Try more File1.txt and you will get an error since you are no longer allowed to read the file.

To view a file’s permissions, type ls –l File1.txt. It will show the permissions for the owner, group, and others

To see this graphically, right-click on the file1.txt in Finder. You will get an information window. At the bottom are the Sharing & Permissions that are currently set.

From the command line, change the file permissions and watch the window change.

chmod u=rw file1.txt

chmod u=r file1.txt
