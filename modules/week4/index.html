# 1. Git

# What is Git

Git is a distributed control system and knowing git is one of the most important skills you would have to have as a developer. Git is not the only version control system existing Today but it is a de-facto standard in the software industry and it is very likely you will find it in many, if not all, projects you participate.

It is far more than a backup system, it tracks all changes made to source code overtime. Also, it allows development collaboration by giving users a set of tools to work together in the same code base.

Since Git records any changes to files (include deletions) you can go back in time and retrieve any version of your file or test few changes here and there before commiting to a final version.

# How does Git work

There are three areas to git. The Working Tree the Staging Area and the Repository.

  • The Working Tree is the area where you are currently working. It is where your files live.
  • The Staging Area is where git starts tracking and saving changes that occur in files.
  • The Local Repository is the area that has everything saved or, as we say, commited.

Your files will ocuppy these areas as you work. Let us see how.

Initially your files are untracked, then you add them to the staging area. Once you are satisfied with the staging, you commit the files to the repo. You can, at any point retrieve a version of your commited files using the checkout command. That is basically, what git does.

# Setup

It is possible that you already have a copy of Git in your Mac. To check, go to you shell and type:

git --version
1

You should see a message showing what is the version you have installed. It may look like this:

git version 2.24.3 (Apple Git-128)
1

If you don't have it already, go to this link download it and install it. After it is installed, try git --version one more time.

Ok, now that you have git installed, let us configure your name and email. Here is what you need to do:

git config --global user.name "Your Name"
1

and

git config --global user.email "your@email.com"
1

We do this so Git can keep track of not only what changes were made but also who made the changes. This is specially useful when you are working in collaboration with other developers

# Initializing a repo

Repo, the short for repository, is a container for a project that is tracked by Git.

There are two types of repos.

  • Local repository - A repository stored on your own machine. It allows youto work on the local version of a project.

  • Remote repository - It usually resides on a remote server offering an "off-site backup" to your files. It's especially relevant when working in teams since it is the place where you can share your work and see other people's code.

Right now you have a folder with some files in it. It is a project, but it is not yet a repo. To create a new repository (and to start to track the files), go to the main folder of your project and type:

git init
1

TIP

The default branch name is master. If you want to use another name for it, at the moment of initialization (in version 2.28), you can use git -i <anotherName>

That will create a hidden .git directory for your project and that is where Git saves all info it need to track the repo. If you delete this directory you are effectively removing the repo (but not the files) from this project.

You shoud see something like

Initialized empty Git repository in /Users/YourName/Documents/code/mad9112/education/.git/
1

If so, great. We are on track. Literally.

# Exercising your git powers

To practice some git-fu, let us do the following:

  1. Open index.html in your favorite editor.
  2. Change the www.yourdomainname.com to www.yourname.com and save the file.
  3. Test the site to see if your changes are effectively.
  4. Commit your changes.
  5. Check the history (git log).
  6. On index.html change the phone area code and save it.
  7. Check the site to see if your changes worked.
  8. DO NOT commit just yet.
  9. Open courses.html and fix the typo (It should read courses in the plural and not course)
  10. Save these changes.
  11. Stage and commit courses.html
  12. Stage and commit index.html.
  13. Open pricing.html.
  14. Change the Silver plan to be $99 per year and save it.
  15. Test to see if the change worked.
  16. Commit your changes.
  17. Change the term PLAN & PRICING to PLANS on any file it appears.
  18. Save and test the changes.
  19. Commit.
  20. Check the log
  21. Again, in index.html remove all lines after line 100.
  22. Without testing it first, commit ìndex.html
  23. Checkout the initial version of the index.html and commit.
  24. Change the title of this page to "Great Courses"
  25. Save and test the changes.
  26. Commit the changes.
  27. Delete all html files except index.html and commit these changes.

You have broken you site. Test it and you will see it. Fix it! 😬
