# 1. The shell

Shell is the interpreter for the command line interface. Unix and therefore, macOS, has many available shells available. Each have a differente story and set of features. All of them share the same set of commands and basic functionality. They have names like Bourne, Bash, fish, C, csh, tcsh and so on. When open your Teminal to use the CLI, you are in a shell.

macOS Mojave and previous versions used Bash as their default shell, but in 2019 Apple decided to change that and set the Z shell as their new default.

Z shell (or just Zsh) , is an extended version of the Bourne shell (Bash), with plenty of new features, and support for plugins and themes. Since it‚Äôs based on the same shell as Bash, Zsh has many of the same features.

# 4. Environment variables in macOS

An environment (ENV) variable is a variable that you create, just like in JavaScript, which can be read and used by the Operating System. Every OS has ENV variables.

These variables can be accessed, not just by the OS but also by other programs running on the OS. They can be used to share settings and information between programs. Useful commands to use are export, echo and printenv.

  • export makes the variable available to the environment
  • echo outputs a string to the console
  • printenv print the values of the enviroment variables

# 5. PATH variable

There is one ENV variable that is used very frequently. It is called PATH.

PATH is used as a list of places where programs are installed. If, in the Terminal, you type the name of a program and hit enter/return, then the OS needs to try and find that program to start it running.

It should look inside the current folder first but then you can't search every folder on the storage drive for a program with the correct name. THIS is why we have the PATH. It is the list, in order, of the places to search for the program.

The programs could be ones that are part of the OS or they could be things like VSCode, which you have installed yourself.

Between each path part you need to add a : colon.

You should wrap the whole value inside quotation marks to avoid problems caused by spaces in the path.

Activity - to add the PATH variable to the .zshrc file.

export NODE_PATH="/Users/<username>/.npm-packages"
export VSCODE="/Applications/Visual Studio Code.app/Contents/Resources/app"

export PATH=".:/bin:/sbin:/usr/bin:/usr/sbin:/usr/local/bin:$NODE_PATH/lib/node_modules:$NODE_PATH/bin:$VSCODE_HOME/bin"
